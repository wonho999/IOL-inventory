## 🎯 핵심 기획 — "시트가 진실의 원천(SSOT)"

### 데이터 출발점
스크린샷의 시트 한 줄이 모든 것의 시작:

```
| 렌컴일 | 고객이름 | 수술일 | 종류    | 도수 | CYL | 눈 | 주문자 | 수령자 |
| 5/4   | 최원호  | 5/7   | 오디세이 | 21.0 | -  | OD | 정승기 | 이은혜 |
```

### 렌즈 라이프사이클 = 컬럼이 채워지는 순서

```
[1] 렌컴일 입력      → 🟡 결정됨 (렌즈 필요 확정, 도수/CYL 알게 됨)
[2] 주문자 입력      → 🟠 발주됨 (주문 진행)
[3] 수령자 입력      → 🔵 입고됨 (재고 +1, 수술 예정 확정)
[4] 수술일 도래      → 🟢 수술중 (오늘 수술)
[5] 수술일 < 오늘   → ⚫ 사용완료 (재고 -1, 통계 누적)
```

### 상태 자동 판별 (코드 레벨)

```javascript
function getRowStatus(row, today) {
  if (!row.lensName || !row.diopter) return 'incomplete';
  if (!row.orderedBy)               return 'decided';     // 🟡
  if (!row.receiver)                return 'ordered';     // 🟠
  if (row.surgDate > today)         return 'received';    // 🔵
  if (row.surgDate === today)       return 'in_progress'; // 🟢
  return 'used';                                           // ⚫
}
```

### 재고 = 시트에서 자동 산출

- 별도 재고 입력 화면 없음
- `received(🔵) + in_progress(🟢)` 행을 모델/도수별로 카운트 = 현재 재고
- `used(⚫)` 행은 누적 사용 통계로

### 입력 분리

- **시트(구글)**: 입력 도구 (실무자 익숙, 동시 편집)
- **앱**: 시각화 + 알림 도구 (재고 셀, 일정, 통계, 부족 경고)

## 🗂 새 탭 구성

기존 4개 탭(대시보드/재고/주문현황/구글시트연동/CRM스캔) → **5개로 명확하게 재구성**

```
🏠 홈 / 대시보드
   - 오늘의 수술 일정 (입고 OK / 미입고 강조)
   - 이번 주 시각화
   - 재고 부족 경고
   - 주문 대기 / 입고 대기 카운트

📊 주문 시트 (메인 탭, 기본 진입점)
   - 구글 시트 실시간 동기화 / 직접 입력
   - 엑셀 스타일 편집 가능 테이블
   - 행 상태 색상 자동 표시
   - 미완성 데이터(도수/CYL 누락) 빨간 경고

📦 재고 현황
   - 시트에서 자동 산출된 모델/도수별 그리드
   - 실시간 재고 셀 + (예정 N) 표시
   - 직접 편집 불필요 (시트에서만 입력)
   - 보기/검색/필터 위주

📈 통계
   - 월별 사용량
   - 모델별 회전율
   - 인기 도수 TOP
   - 주문→입고 평균 소요일

⚙️ 설정
   - 렌즈 모델 관리 (도수 직접 입력)
   - 구글 시트 URL / 수령자 이름 등록
   - 화면 텍스트 / 탭명 변경
   - 백업/복원
```

## 🛠 작업 시 지킬 것

1. **상태 변수는 스크립트 최상단에 const/let**
2. **localStorage 키는 항상 `iol3_` prefix**
3. **HTML 텍스트는 data-label 속성 사용**
4. **변경 후 적절한 save() 함수 호출**
5. **node --check 로 script 블록 문법 검증**
6. **개인정보(이름/차트번호)는 코드에 하드코딩 금지** — 익명 더미만 OK
7. **각 함수는 한 가지 책임만** — 기존 코드의 여러 곳에서 같은 일 처리하던 부분 정리

## 🚀 작업 우선순위 (새 대화에서 진행 순서)

### Phase 1: 기반 설계 (1차 작업)
1. 새 데이터 구조 (`SHEET_DATA` 단일 소스)
2. 상태 자동 판별 로직 (`getRowStatus`)
3. 시트 → 재고 자동 산출 (`computeStock`)

### Phase 2: 핵심 기능
5. 주문 시트 탭 (엑셀 스타일 편집 + 상태 색상 + 구글 시트 동기화)
6. 재고 현황 탭 (자동 산출, 그리드, 우클릭 메뉴)
7. 홈 대시보드 (오늘 + 주간 + 경고)

### Phase 3: 보조 기능
8. 통계 탭 (월별/모델별)
9. 설정 탭 (렌즈 모델, 구글 시트, 수령자, 라벨)
10. 모바일 최적화

### Phase 4: 마무리
11. 익명 공개 버전 (`index_public.html`)
12. GitHub 배포
